# Complee

Guided regulatory expansion playbook for fintechs (PSD3, GDPR, DORA, and more).

## Tech Stack

- **Framework**: TanStack Start v1 (React 19 + Vite 7)
- **Styling**: Tailwind CSS v4 (configured via `src/styles.css`)
- **Backend**: Lovable Cloud (Supabase)
- **Routing**: TanStack Router (file-based, in `src/routes/`)
- **State**: Zustand stores in `src/store/`
- **Deploy target**: Cloudflare Workers (see `wrangler.jsonc`)

## Getting Started

```bash
bun install
bun run dev
```

The app will start on `http://localhost:3000`.

### Build

```bash
bun run build
```

### Type-check

```bash
bunx tsc --noEmit
```

## Environment Variables

The `.env` file is auto-generated by Lovable Cloud and contains:

- `VITE_SUPABASE_URL`
- `VITE_SUPABASE_PUBLISHABLE_KEY`
- `VITE_SUPABASE_PROJECT_ID`

If you self-host, recreate `.env` with values from your own Supabase project.

## Database

Schema migrations live in `supabase/migrations/`. The TypeScript schema is mirrored in `src/integrations/supabase/types.ts` (auto-generated).

Tables:
- `profiles` — user profile metadata
- `assessments` — saved assessment runs
- `step_progress` — per-step progress and form inputs

All tables have RLS enabled and are scoped by `user_id`.
